Case Study for Meet-App

Overview

To build a serverless, progressive web application (PWA) with React using a test-driven development (TDD) technique. It uses AWS Lambda, OAuth2 through Google, and the Google Calendar API. It shows all the events from a public Google Calendar and allows the user to filter for specific cities.

Key Features

Key features for this project are:

  1. Filter events by city.
  2. Show/hide event details.
  3. Specify number of events.
  4. Use the app when offline.
  5. Add an app shortcut to the home screen.
  6. View a chart showing the number of upcoming events by city.

Approach

Here I have done it in two parts. First part is to get access from Google Calendar API. The other part is to display the events from the API.

To get the access from Google you need secret credentials for authentication and the Token. For the credentials I have used OAuth, which gives a code in the url after authentication. Using this code, we can get Token using an AWS server. I have set up auth-server and added serverless access key from AWS console. Then I added config file to connect to the google. And I saved the token in a cookie, to get data from API.

To show the events from API, I have used test-driven and behavior-driven development. I have used Jest for Unit testing and Integration testing. Then I have used Cucumber for Acceptance testing. And I have used Puppeteer for End-to-End testing.
