Case Study for myFlix App using Angular

Overview

To build a web application called MyFlix app using Angular. The data is based on its existing server-side code (REST API and database), with supporting documentation. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Purpose and Context

MyFlix was a personal project I built as a part of my web development course at CareerFoundry. My role was to build a web application’s complete server side, including the application’s server, business logic, and business layers. And to build a single-page, responsive application with routing, rich interactions, several interface views, and a polished user experience based on server-side code(REST API and Database).

Duration

It took me over 4 weeks to complete the project. Learning the concepts of Angular project structure, and routing, and applying them to a project took me more time.

Approach

1. Server-side Movies application

I created a Restful API using Node JS and Express framework that interacts with the non-relational database MongoDB. The API can be accessed via commonly used HTTP methods like GET, POST, PUT or DELETE. To retrieve data and store data from the database CRUD methods are used. The API provides movie information in JSON format. To test the API, I used Postman. I also included user authentication and authorization using HTTP authentication and JWT authentication. Finally, I hosted the API using Heroku and connected to Mongo Atlas using mongoose.

2. Client-side MyFlix application

After completing the API, I started to build the interface where users can make requests, and receive responses from the server side. Its a single-page, responsive movie app built with Angular, with routing and several interface views. The app opens with a Welcome Page, where users will be able to either log in or register an account. Once authenticated, the user should now view all movies. A single Movie view displays movie Title and a director name, on clicking the director you get a dialog box which displays the details of the director. Then you have Genre, Synopsis for the movie, on clicking you get respective details. You can even favorite the movie. And you have Profile page where you can update the user information.
